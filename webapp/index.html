04

System Architecture

Built around composition over inheritance: the engine is built from interchangeable component objects, not a monolithic class.

TurbojetEngine composes ↓run_cycle()
Inlet(efficiency) → ram recovery
Compressor(efficiency, pressure_ratio) → isentropic compression
Combustor(efficiency, fuel_heating_value, exit_temp) → energy balance
Turbine(efficiency) → spool-balanced expansion
Nozzle(efficiency) → choked / fully-expanded flow
ParameterCalibrator abstract base ↓calibrate()
OptimizationCalibrator — scipy L-BFGS-B least squares
SurrogateMLCalibrator — Random Forest trained on physics-generated data
ABSTRACTION

EngineComponent ABC

Every component implements the same process() interface — new component types plug in with zero changes to TurbojetEngine.

COMPOSITION

Built from parts

TurbojetEngine holds component instances rather than inheriting from them — swap a HighBypassFan in for Compressor freely.

POLYMORPHISM

Interchangeable calibrators

OptimizationCalibrator and SurrogateMLCalibrator share one interface — the Calibration Lab above calls whichever one you pick, live, through the Flask API.
